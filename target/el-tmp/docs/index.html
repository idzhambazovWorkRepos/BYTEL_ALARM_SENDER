<State the general purpose of the node under the heading. Example:>

EventLink Backup Node is used for backing up collected and distributed event records in Comptel EventLink.

Functionality of EventLink <name of the node> Node <Example: Functionality of EventLink Backup Node>

<Provide a summary of the functionality. Example:>

Backup Node is used for backing up files running in a process stream to a local host or to a remote host with FTP or Secure FTP (SFTP) protocol. The backing up process is implemented as parallel backup, where a copy of the output files from the previous node is configured from UI to be sent to the Backup Node, which then backs up the data.

You can define the following:

�         If the backed-up files are compressed or not.

�         If, in addition to the original backup file, a new file with the same name is created. The new file name gets a suffix and the file contains the backed-up file. The new file is stored in the destination directory after a successful transfer.

�         The backed-up files are deleted after a certain time or when the number of files exceeds a certain limit.

<Give a suitable heading for the functionality><Example: Backing up files>

<Provide a description of the functionality. Example:>

The input data for the Backup Node can be in any format. Backup Node creates a backup of all received files copies in the destination host and target directory and with the specified method.

Backup Node has the following compression options for the backed up files before transfer:

�         bzip2

�         gzip

�         compress

�         no compression

You can select the option from a drop-down list.

At startup, Backup Node checks the availability of the selected compression program, first in the path and then in a standard location for the program. If the compression program is not found, an error message is sent to node base and the execution is aborted.

If the compression of a file fails during processing, for example, because there is not enough memory, an error message shows the reason for the failure and Backup Node fails. After this, Node Manager tries to restart it three times.

You can define the life time or the number of backed-up files that can be stored in the destination directory, or both, before they are deleted.

<Give a suitable heading for the functionality><Example: Reprocessing backed-up files>

<Provide a description of the functionality. Example:>

When backed-up files need to be reprocessed, EventLink FTP Collector in local collect mode is used. In order to be able to access the original files, you must manually uncompress the backed-up files and put them into the polled directory, where the system automatically processes them.

<Give a suitable heading for the functionality><Example: Differences between FTP and SFTP protocols>

<Provide a description of the functionality. Example:>

The SFTP protocol supports only a subset of the FTP protocol functionality. The following list sums up the differences between the protocols:

�         SFTP supports binary mode only.

�         With SFTP, active and passive connection types are not relevant.

�         With SFTP, transfer block sizes are not relevant.

<Next, list the supported encryption algorithms under the heading below.>

Supported encryption algorithms

<Provide a description of supported encryption algorithms. Example:>

In SFTP mode, Backup Node supports the DSS public key encryption method. The supported ciphers for encryption are DES, DES3, Blowfish and RC4. Ensure that the remote SSH server has been configured to support one of these encryption methods.

Configuration examples

<List the most important examples. Example:>

Example 1:

The following gives an example configuration when the backed-up files are not compressed in local mode. The number of files stored in the destination directory is limited to 10 000 and they are stored for 30 days.

CompressionMethod no compression

MaxFiles 10000

MaxDays 30

Ruleset back_up.pm

TargetDirectory /local/data

DestinationHost localhost

TransferProtocol Local

FlagFileSuffix <empty>

Username backup node

Password backup node

 

Example 2:

The following gives an example configuration when the backed-up files are compressed in FTP mode. The number of files stored in the destination directory is limited to 10000 and they are stored for 30 days.

CompressionMethod .bzip2

MaxFiles 10000

MaxDays 30

Ruleset back_up.pm

TargetDirectory /remote/data

DestinationHost host.comptel.com

TransferProtocol FTP

FlagFileSuffix <empty>

Username backup node

Password backup node

 

Example 3:

The following gives an example configuration when an unlimited number of backed-up files are cleaned from the destination directory after 5 days in FTP mode. The flagfile is not created.

CompressionMethod .bzip2

MaxFiles 0

MaxDays 5

Ruleset back_up.pm

TargetDirectory /remote/data

DestinationHost host.comptel.com

TransferProtocol FTP

FlagFileSuffix <empty>

Username backup node

Password backup node
